GAI Token服务中枢 GAI Token Service Hub

为全院师生提供统一、合规、可管控的生成式 AI（GAI）服务，一次接入，统一调度。

系统运行中 入口 · 47.83.118.57:3000 兼容 OpenAI API
01

这是什么

本中枢是学院统一建设的生成式人工智能（GAI）服务平台。它把多家厂商的大模型（对话、代码、推理、向量等能力）汇聚到一个入口，按账号分配访问令牌（Token）与使用额度，供全院师生用于教学、科研与创新实践。

过去，老师和同学各自申请、各自付费购买不同平台的 API，存在额度浪费、难以管控、敏感数据外流等问题。中枢的目标是一次接入、统一调度：

// 01

一个入口

用一套地址和令牌即可调用多种模型，无需分别注册各家平台。

// 02

一份额度

学院统一采购、按需分配，用量公开透明，可在控制台随时查看。

// 03

一道防线

调用集中经过学院网关，便于审计与合规管理，降低数据外流风险。

02

三步开始使用

01

登录系统

访问 http://47.83.118.57:3000/，使用学院指定的用户名和密码登录。

02

创建你的访问令牌（API Token）

登录后进入 「令牌」(Token) 页面，点击「添加令牌」。可为每个用途单独创建一个令牌（例如「课程作业」「毕业设计」「科研项目」各一个），并按需设置额度上限和过期时间。

令牌格式形如 sk-xxxxxxxxxxxxxxxxxxxx，等同于你的账号密码，请妥善保管。
03

发起调用

本中枢完全兼容 OpenAI API 格式。你只需把原有代码里的请求地址（base_url）指向中枢，把密钥换成你的令牌即可，其余代码无需改动。

Python · OpenAI SDK 
from openai import OpenAI

client = OpenAI(
    base_url="http://47.83.118.57:3000/v1",   # 指向本中枢
    api_key="sk-你的令牌",                # 切勿硬编码，建议读环境变量
)

resp = client.chat.completions.create(
    model="模型名，如 gpt-4o-mini",
    messages=[{"role": "user", "content": "你好，请做个自我介绍"}],
)
print(resp.choices[0].message.content)
cURL 
curl http://47.83.118.57:3000/v1/chat/completions \
  -H "Authorization: Bearer sk-你的令牌" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "模型名",
    "messages": [{"role": "user", "content": "你好"}]
  }'

04

额度与用量

  • 每个账号拥有学院分配的额度（Quota），调用按所用模型的 token 量与计费倍率扣减。
  • 在控制台的 「数据看板」/「日志」 中可查看实时余额、历史用量与每次调用明细。
  • 额度不足或令牌到期会导致调用失败（返回 quota / expired 相关错误），可在控制台申请续额或新建令牌。
05

使用规范与数据安全

本中枢面向教学科研开放，请共同维护合规、安全的使用环境。
  • 令牌即密码：不要泄露、不要写死在源码里、不要提交到 Git 仓库。建议通过环境变量读取。
  • 数据合规：严禁将涉密信息、学生个人隐私（成绩、身份证、联系方式等）、未公开的科研数据上传给第三方模型处理。
  • 学术诚信：模型输出仅供辅助，课程作业与论文写作须遵守学院学术规范，不得直接抄袭生成内容。
  • 合理使用：额度为公共资源，请避免无意义的批量请求；批量任务请控制并发，必要时与管理员沟通。
  • 警惕注入：当让模型处理外部网页、文档、邮件等内容时，注意其中可能夹带的恶意指令，核对输出后再执行。
